# YouSells UI 优化方案

> 制定时间：2026-05-24
> 目标：从"能用"到"好用且好看"，建立可复用的设计系统

---

## 一、现状诊断

### 1.1 项目结构
- **16 个页面视图**，覆盖 CRM 核心流程（客户→跟进→任务→报告→协作）
- **大量使用 Element Plus** 组件（按钮 122 次、标签 76 次、表格 35 次、表单 116 项）
- **已有基础设计系统**：`base.css` 定义了颜色/间距/圆角/阴影 Token，暗色模式已初步支持
- **布局稳定**：Sidebar + Header + Content 三栏布局，移动端有底部导航

### 1.2 核心问题

| 问题 | 影响 | 严重程度 |
|------|------|---------|
| **Element Plus "默认感"过重** | 大量页面像"开箱即用"的后台模板，缺乏品牌辨识度 | 🔴 高 |
| **信息层级不突出** | 表格、表单、卡片之间视觉权重相近，用户难以快速定位重点 | 🔴 高 |
| **缺少过渡动效** | 路由切换、数据加载、交互反馈都是瞬间跳变，体验生硬 | 🟡 中 |
| **数据可视化薄弱** | Dashboard 只有 2 个基础图表，大量数据以纯文本呈现 | 🟡 中 |
| **移动端适配粗糙** | 有移动端导航但页面内容未针对小屏优化 | 🟢 低 |
| **空状态/异常状态简陋** | 多数页面只有 `el-empty` 或简单文字提示 | 🟢 低 |

---

## 二、优化目标

1. **建立统一的设计语言** — 让所有页面看起来像"同一个产品"
2. **提升信息可读性** — 通过视觉层级引导用户注意力
3. **增加情感化设计** — 适当的动效、插画、微交互提升体验
4. **强化数据表达** — 让数字会说话，降低认知负担
5. **完善边缘场景** — 空状态、加载中、网络异常都有体面处理

## 三、设计系统规范（Design Token）

> 基于现有 `base.css` 扩展，不推倒重来

### 3.1 颜色系统（已具备，需补充语义色）

```css
/* 新增语义颜色变量 */
--color-bg-elevated: /* 浮层背景（下拉框、tooltip） */
--color-overlay: /* 遮罩层 */
--color-divider: /* 分割线 */

/* 数据可视化色板 */
--chart-color-1: #2563eb;
--chart-color-2: #10b981;
--chart-color-3: #f59e0b;
--chart-color-4: #ef4444;
--chart-color-5: #8b5cf6;
--chart-color-6: #06b6d4;
```

### 3.2 间距系统（现有较散，需统一）

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | 4px | 图标与文字间距、紧凑内联 |
| `--space-sm` | 8px | 按钮内部、小卡片间距 |
| `--space-md` | 12px | 表单元素间距 |
| `--space-lg` | 16px | 卡片内部 padding |
| `--space-xl` | 20px | 模块之间间距 |
| `--space-2xl`| 24px | 页面 Section 间距 |
| `--space-3xl`| 32px | 大模块分隔 |

### 3.3 动效规范（新增）

```css
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);
--ease-in-out-sine: cubic-bezier(0.37, 0, 0.63, 1);

--duration-fast: 150ms;   /* hover、按钮反馈 */
--duration-base: 250ms;   /* 展开、切换 */
--duration-slow: 350ms;   /* 页面过渡、弹窗 */
```

### 3.4 字体层级（新增）

| 层级 | 大小 | 字重 | 用途 |
|------|------|------|------|
| Display | 24px | 700 | 页面大标题 |
| H1 | 20px | 700 | Section 标题 |
| H2 | 16px | 600 | 卡片标题 |
| H3 | 14px | 600 | 子标题、表格列头 |
| Body | 14px | 400 | 正文 |
| Caption | 12px | 400 | 辅助文字、时间戳 |

## 四、分阶段实施计划

### 📌 Phase 1：设计系统地基（1-2 天）
**目标：统一所有页面的基础视觉语言**

| 任务 | 改动范围 | 验收标准 |
|------|---------|---------|
| 1.1 扩展 base.css Token | `src/styles/base.css` | 间距/动效/字体/语义色变量全部定义完成 |
| 1.2 统一卡片组件 | 所有 `.page-card` 使用场景 | 所有页面卡片圆角、阴影、padding 一致 |
| 1.3 统一按钮风格 | 全局 `el-button` | 主按钮、文字按钮、危险按钮风格统一 |
| 1.4 统一空状态 | 各页面的 `el-empty` | 自定义空状态插画 + 文案，替代默认空状态 |
| 1.5 统一加载状态 | 全局 `v-loading` | 加载动画风格统一，表格加载加骨架屏 |

### 📌 Phase 2：核心页面视觉升级（3-4 天）
**目标：让客户管理和看板成为"门面担当"**

#### 2.1 首页看板 Dashboard（优先级最高）

**现状问题：**
- 欢迎区蓝色渐变 banner 比较突兀
- 统计卡片是纯数字，缺乏趋势指示
- 图表只有折线图和饼图，信息维度少
- "待办事项"区域只是简单列表

**优化方案：**
```
┌─────────────────────────────────────────────────────┐
│  👋 欢迎区（缩减高度，更精致）                       │
├─────────────────────────────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐         │
│  │ 客户 │ │ 跟进 │ │ 成交 │ │ 任务 │ │ 转化 │ ← 趋势箭 │
│  │ 128 │ │ 42  │ │ 15  │ │ 8/10│ │ 32% │   头 + 环比  │
│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘         │
├─────────────────────────────────────────────────────┤
│  ┌─────────────────────┐  ┌─────────────────────┐ │
│  │   跟进趋势（面积图）  │  │  客户阶段分布        │ │
│  │   近30天折线+面积    │  │  漏斗图替代饼图      │ │
│  └─────────────────────┘  └─────────────────────┘ │
├─────────────────────────────────────────────────────┤
│  ┌─────────────────────┐  ┌─────────────────────┐ │
│  │  🔔 待跟进客户      │  │  😴 沉默客户（3天）  │ │
│  │  卡片列表+快捷操作   │  │  卡片列表+快捷操作   │ │
│  └─────────────────────┘  └─────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

**具体改动：**
- 统计卡片增加"环比变化"小标签（↑ 12% / ↓ 5%）
- 折线图升级为面积图，增加「近7天/近30天/本季度」切换
- 饼图替换为漏斗图，直观展示客户转化漏斗
- 新增「沉默客户预警」模块（红黄绿三级预警）
- 待办任务增加「一键创建跟进」快捷入口

#### 2.2 客户管理 CustomerList（核心功能）

**现状问题：**
- 表格信息密度高但重点不突出
- 筛选栏按钮排列较乱
- 缺少客户状态可视化

**优化方案：**
- 表格行增加「客户健康度」彩色竖条（绿色=活跃，黄色=需跟进，红色=沉默）
- 客户名字段增加头像+名字组合，hover 显示快捷操作（打电话、发微信、写跟进）
- 意向度用「进度条」替代纯文字标签
- 筛选栏收纳为「高级筛选」抽屉，默认只显示搜索框
- 增加「卡片视图」切换按钮（表格 ↔ 卡片网格）

#### 2.3 客户详情 CustomerDetail（高频页面）

**现状问题：**
- 档案卡片和元数据面板视觉权重相近
- 跟进时间轴缺少时间筛选
- 操作按钮分布零散

**优化方案：**
- 顶部增加「客户阶段进度条」（线索 → 邀约 → 试听 → 成交）
- 档案卡片突出显示「下次跟进时间」（临期红色高亮）
- 时间轴增加「按类型筛选」（电话/微信/面谈/其他）
- 底部增加「快捷跟进」浮动按钮

### 📌 Phase 3：协作与报告场景优化（2-3 天）

#### 3.1 任务板 TaskBoard

**现状问题：**
- 看板列是简单列表，拖拽体验一般
- 任务卡片信息展示单一

**优化方案：**
- 任务卡片增加「优先级角标」（紧急=红色，高=橙色，普通=灰色）
- 卡片显示负责人头像缩略图
- 拖拽时增加「拖拽阴影」和「放置区域高亮」
- 增加「泳道视图」切换（按负责人分组）

#### 3.2 报告广场 ReportPlaza

**现状问题：**
- 报告卡片是文字为主，阅读疲劳
- 缺少互动反馈

**优化方案：**
- 卡片顶部增加「作者头像 + 发布时间」行
- 报告内容用「emoji + 关键词」高亮（✅ 完成 / ⚠️ 问题 / 📅 计划）
- 点赞/评论增加微动画（心形跳动、数字滚动）
- 增加「只看我的」快捷筛选

### 📌 Phase 4：动效与交互体验（2 天）

| 任务 | 描述 |
|------|------|
| 4.1 路由过渡动画 | 页面切换时淡入淡出 + 轻微上移 |
| 4.2 表格行动画 | 新增行「滑入」、删除行「淡出」 |
| 4.3 弹窗动画 | 从中心缩放淡入，关闭时反向 |
| 4.4 按钮反馈 | 点击时涟漪效果（Ripple） |
| 4.5 数字动画 | 统计数字从 0 滚动到目标值 |
| 4.6 骨架屏 | 表格/卡片加载时显示骨架屏替代转圈 |

### 📌 Phase 5：边缘场景 & 移动端（2 天）

| 任务 | 描述 |
|------|------|
| 5.1 全局空状态 | 每个模块定制插画 + 引导操作（如"暂无客户，去创建第一个客户 →"） |
| 5.2 全局异常处理 | 网络错误、权限不足、数据为空都有统一错误页 |
| 5.3 移动端表格 | 表格转为卡片列表，横向滑动查看详情 |
| 5.4 移动端表单 | 增大触控区域，底部固定提交按钮 |
| 5.5 触摸手势 | 左滑删除、下拉刷新 |

## 五、关键组件设计稿

### 5.1 统计卡片（StatCard）

```
┌─────────────────────────────┐
│  👤 客户总数      ↑ 12%    │  ← 右上角趋势标签
│                             │
│         128                 │  ← 大号数字，滚动动画
│      较上周 +14            │  ← 副文案
└─────────────────────────────┘
```

**Props：** `title`, `value`, `trend` (↑/↓/null), `change`, `icon`, `color`

### 5.2 客户健康度条

```
┌──┬─────────────────────────────┐
│🟢│  张三 | T1 | 很稳 | 初中    │  ← 行首彩色竖条
```

- 🟢 绿色：7 天内有跟进
- 🟡 黄色：3-7 天未跟进

## 六、技术实现要点

### 6.1 新增依赖建议

| 库 | 用途 | 体积 |
|----|------|------|
| `vue-countup-v3` | 数字滚动动画 | 极小 |
| `@vueuse/motion` | 滚动触发动画 | 小 |
| 无需新增 | 路由过渡用 Vue `<Transition>` | 内置 |

### 6.2 代码组织建议

```
src/
  styles/
    base.css          ← 现有，继续扩展 Token
    animations.css    ← 新增，全局动效关键帧
    dark-mode.css     ← 新增，暗色模式覆盖（从 base.css 拆分）
  components/
    ui/               ← 新增，设计系统组件
      StatCard.vue
      EmptyState.vue
      HealthBadge.vue
      SkeletonTable.vue
      TrendTag.vue
    ...
```

### 6.3 兼容性注意

- 所有动效使用 `prefers-reduced-motion` 媒体查询，尊重用户「减少动效」设置
- 暗色模式覆盖继续采用 `[data-theme="dark"]` 方案，保持现有架构
