### 核心业务逻辑

```
getAiInsight(id):
  1. customerMapper.selectById(id) — 客户存在性 + 数据权限校验
  2. 若 !forceRefresh: aiInsightCacheMapper.selectValidByCustomerId(id) → 命中返回
  3. queryFollowUps(id, limit 20) → 空则返回 AiInsightResponse.empty(id)
  4. buildUserPrompt(client + followups) → aiService.generateStructured()
  5. enrichResponse() 填充 null 默认值
  6. saveCache() → 先 deleteByCustomerId 再 insert（避免唯一键冲突）
  7. 返回响应

refreshAiInsight(id):
  同 getAiInsight，但 forceRefresh=true，跳过步骤 2

createFollowUp():  // FollowUpServiceImpl
  insert 跟进 → selectValidByCustomerId → 置 expiresAt=NOW → updateById
```

### 缓存失效链

```
创建跟进 → FollowUpServiceImpl.createFollowUp()
        → aiInsightCacheMapper.selectValidByCustomerId(customerId)
        → 有缓存 → expiresAt = now() → updateById (标记过期)
        → 无缓存 → 跳过

下次 GET → selectValidByCustomerId(customerId, expires_at > NOW())
        → 已过期 → 返回 null → 重新生成 + 覆盖旧缓存
```
