# 开发分工与首期推进

## 1. 开发团队

当前开发团队共 `5` 人：

- `秦梓源`：队长，后端主负责人
- `志明`：后端
- `哲涛`：后端
- `嘉诚`：前端
- `许润`：前端

## 2. 当前建议分工

### 后端组

- `秦梓源`
  负责整体架构、数据库设计、权限模型、核心接口、部署上线。
- `志明`
  负责客户模块、跟进记录模块、日报周报模块。
- `哲涛`
  负责公共安排区、话术库、统计接口、辅助模块。

### 前端组

- `嘉诚`
  负责整体前端框架、页面布局、首页看板、客户总表。
- `许润`
  负责客户详情、跟进记录、公共安排区、日报周报页面。

## 3. 协作原则

- 后端先统一字段和接口口径
- 前端先做 `P0` 页面
- 每个模块必须有明确负责人
- 先打通主流程，再补体验和扩展模块

## 4. 当前建议的第一阶段开发目标

第一阶段不求做全，先打通下面这条主流程：

`登录 -> 首页看板 -> 客户总表 -> 客户详情 -> 跟进记录 -> 公共安排 -> 日报/周报 -> 话术库`

## 5. 第一阶段建议里程碑

### 阶段一：项目基线搭建

- 初始化前后端项目
- 定技术栈
- 定包结构和目录结构
- 定数据库命名规范
- 定接口响应格式
- 定登录与权限基线

### 阶段二：核心数据结构

- 用户与角色
- 客户表
- 跟进记录表
- 公共安排表
- 日报 / 周报表
- 话术库表

### 阶段三：P0 页面和接口联调

- 登录页
- 首页看板
- 客户总表
- 客户详情
- 跟进记录
- 公共安排区
- 日报 / 周报
- 话术库

### 阶段四：测试与上线准备

- 联调
- Bug 修复
- 权限验证
- 部署脚本
- 测试环境验证

## 6. 当前马上要做的事

1. 定权限模型
2. 定数据库表设计
3. 定接口规范
4. 初始化前后端仓库结构
