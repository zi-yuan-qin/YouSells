# 项目需求概览

## 1. 项目定位

`YouSells` 是一个给团队内部使用的 `客户管理 + 协作管理 + 内容沉淀` 平台。

它不是单纯的客户表工具，而是要先把你们当前最影响效率的几件事集中到一个地方：

- 管客户
- 记跟进
- 看谁负责谁
- 看今天该跟谁
- 排公共任务
- 交日报周报
- 沉淀话术

## 2. 目标用户

- `管理员 / 队长`
  看全局、改负责人、看报表、调资源。
- `成员`
  录客户、写跟进、看待办、交日报、用话术。

## 3. 当前业务背景

截至 `2026-05-18`，这个平台服务的是你们当前这套真实业务节奏：

- 现在仍以现有客户推进和高考后准备为主
- 后续会逐步承接新生客户
- 团队内部需要统一客户记录、分工、跟进和复盘
- 平台后续要支撑 `50-100` 人内部使用

所以它不是一次性展示项目，而是要能真正上线、能持续使用的内部系统。

## 4. P0 必做模块

### 4.1 首页看板

首页建议先放这 6 块：

- 今日待跟进客户
- 已逾期未跟进客户
- A 类高意向客户池
- 每人名下客户数
- 最近新增客户
- 今日公共安排

### 4.2 客户总表

必须支持：

- 公共查看
- 搜索
- 分组
- 标签
- 负责人
- 当前阶段
- 最近反馈
- 下一步动作

### 4.3 客户详情页

至少展示：

- 基本信息
- 标签信息
- 当前阶段
- 当前顾虑
- 历史跟进时间线
- 下次动作
- 负责人和协助人

### 4.4 跟进记录模块

每个客户都要能写时间线记录：

- 这次聊了什么
- 客户反馈是什么
- 当前顾虑是什么
- 下一步准备怎么打
- 谁来继续跟
- 记录时间

### 4.5 公共安排区

至少要有：

- 事项
- 类型
- 负责人
- 协助人
- 开始时间
- 截止时间
- 当前状态
- 下一步动作

### 4.6 日报 / 周报

第一版先做：

- 每个人能交日报
- 每周能交周报
- 团队长能看汇总

### 4.7 话术库

先支持分类沉淀：

- 验证消息
- 开场模板
- 邀群模板
- 欢迎语
- 跟进句
- 顾虑处理

## 5. P1 模块

- 月报
- 复盘总结区
- 匿名意见箱

## 6. P2 模块

- 工作聊天室
- 知识分享区
- 博客 / 日常分享区

## 7. 当前最关键的需求原则

- 先做一个团队每天真会用的平台
- 先把 `P0` 做稳
- 先解决客户推进和协作效率问题
- 先保证数据口径统一，再谈页面复杂度
