# 本地基础服务启动指南

## 1. 目标

这份文档用于统一 `YouSells` 项目的本地基础服务环境。

当前先统一两类依赖：

- `MySQL`
- `Redis`

目标不是现在就把整套正式部署做完，而是先保证每个开发成员都能稳定起本地开发环境，避免后续正式接入数据库、缓存、登录态时被环境问题卡住。

## 2. 当前统一方案

团队当前统一采用：

- `Docker Desktop`
- `Docker Compose`
- 项目根目录 `docker-compose.yml`

默认容器：

- `yousells-mysql`
- `yousells-redis`

默认宿主机映射端口：

- `MySQL: 13306`
- `Redis: 16379`

之所以不用常见的 `3306 / 6379`，是为了减少团队成员本机已装数据库或缓存时的端口冲突。

## 3. 启动前准备

成员本机需要具备：

- 已安装 `Docker Desktop`
- 已开启 Docker Engine
- 已安装 `JDK 21`
- 本地磁盘有足够空间拉取镜像

建议先执行：

```powershell
docker --version
docker compose version
java -version
mvn -version
```

如果以上命令能正常返回版本号，并且 `java -version` 与 `mvn -version` 都指向 Java 21，说明基础环境可用。

如果 `java -version` 是 Java 21，但 `mvn -version` 显示 Java 11，说明 Maven 绑定了旧 JDK。需要先切换 `JAVA_HOME` 后再执行 Maven，例如：

```powershell
$env:JAVA_HOME="D:\develop\jdk21"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -version
```

## 4. 一键启动命令

在项目根目录执行：

```powershell
docker compose up -d
```

启动后查看状态：

```powershell
docker compose ps
```

正常情况下应看到：

- `yousells-mysql` 为 `running`
- `yousells-redis` 为 `running`

## 5. 当前默认连接信息

### MySQL

- Host: `localhost`
- Port: `13306`
- Database: `yousells`
- Username: `root`
- Password: `root`

### Redis

- Host: `localhost`
- Port: `16379`

这些默认值已经同步写入：

- 根目录 `.env.example`
- 后端 `yousells-server/src/main/resources/application.yml`

## 6. 数据库初始化机制

当前 `docker-compose.yml` 已挂载：

`yousells-server/src/main/resources/db`

目录下的 SQL 会在 MySQL 第一次初始化数据目录时自动执行：

- `01_schema_p0.sql`
- `02_seed_p0.sql`

也就是说：

- 第一次启动容器时，会自动建库建表并导入基础种子数据
- 如果只是普通重启容器，不会重复清空数据

## 7. 常用命令

### 查看日志

```powershell
docker compose logs mysql
docker compose logs redis
```

### 停止服务

```powershell
docker compose stop
```

### 停止并删除容器

```powershell
docker compose down
```

### 停止并删除容器与数据卷

这个命令会清掉本地 MySQL 和 Redis 数据，只在你确认要重置环境时使用：

```powershell
docker compose down -v
```

### 重建并重新初始化数据库

```powershell
docker compose down -v
docker compose up -d
```

## 8. 校验方式

### 校验 MySQL

```powershell
docker exec -it yousells-mysql mysql -uroot -proot -e "SHOW DATABASES;"
```

```powershell
docker exec -it yousells-mysql mysql -uroot -proot -D yousells -e "SHOW TABLES;"
```

### 校验 Redis

```powershell
docker exec -it yousells-redis redis-cli ping
```

如果返回：

- MySQL 能看到 `yousells`
- Redis 返回 `PONG`

就说明基础服务正常。

## 9. 和后端联动时的环境变量

后端默认读取：

- `MYSQL_HOST`
- `MYSQL_PORT`
- `MYSQL_DATABASE`
- `MYSQL_USERNAME`
- `MYSQL_PASSWORD`
- `REDIS_HOST`
- `REDIS_PORT`

当前推荐本地开发值：

```env
MYSQL_HOST=localhost
MYSQL_PORT=13306
MYSQL_DATABASE=yousells
MYSQL_USERNAME=root
MYSQL_PASSWORD=root
REDIS_HOST=localhost
REDIS_PORT=16379
WEB_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

## 10. 团队协作建议

当前阶段建议这样使用：

- 每个开发者本地各自启动一套 `MySQL + Redis`
- 大家不要日常共用同一套开发数据库
- 联调时再接入一套公共测试环境

这样做的好处是：

- 不容易互相污染数据
- 不容易因为某个人误删表影响全队
- 排查问题更容易定位

## 11. 常见问题

### 1. 端口被占用

如果 `13306` 或 `16379` 被占用：

- 先查看本机是否已有其他服务占用
- 如果必须改端口，就同时改：
  - `docker-compose.yml`
  - `.env.example`
  - 本地实际运行环境变量

不要只改一处。

### 2. SQL 没有自动执行

常见原因：

- MySQL 数据卷已经存在
- 容器不是第一次初始化

这时如果你确实需要重新导入初始化 SQL，执行：

```powershell
docker compose down -v
docker compose up -d
```

### 3. Docker Desktop 没启动

表现通常是：

- `docker compose up -d` 直接报错
- 提示无法连接 Docker daemon

先启动 Docker Desktop，再重新执行命令。
