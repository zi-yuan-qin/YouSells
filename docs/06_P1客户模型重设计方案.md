# P1 客户模型重设计方案

日期：`2026-05-21`
基于与秦梓源的详细讨论，重新设计客户模型以匹配真实学生业务场景。

---

## 1. 业务背景

- 目标客户：南京信息工程大学在校学生
- 业务模式：T0/T1 邀约 → T2/T3 沟通洽谈
- 沟通链路：职业规划 → 技术栈 → 课程推荐
- 90% 情况 T0/T1 做邀约、T2/T3 做沟通；10% 自己约自己聊

## 2. 客户核心字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `real_name` | VARCHAR(64) | 真实姓名 |
| `grade` | VARCHAR(16) | 年级（大一/大二/大三/大四） |
| `major` | VARCHAR(128) | 专业 |
| `class_name` | VARCHAR(64) | 班级 |
| `inviter_user_id` | BIGINT | 邀约人 ID |
| `owner_user_id` | BIGINT | 负责人 ID |
| `progress` | VARCHAR(16) | 进度：职规 / 技术栈 / 课程 |
| `intent` | VARCHAR(8) | 意向：很稳 / 可跟 / 观望 / 冷淡 |
| `inviter_note` | TEXT | 邀约备注（邀约人一次性填写，覆盖来源场景、初次印象、关键背景） |

## 3. 进度流转（可回退）

```
职规 ──→ 技术栈 ──→ 课程
  ↑         ↑        ↑
  └─────────┴────────┘  （允许回退）
```

- 允许回退：聊到技术栈发现职规没聊透，可以回到职规阶段
- 意向与进度独立：进度是"聊到哪了"，意向是"成功可能性"

## 4. 双字简码意向

| 简码 | 含义 |
|---|---|
| 很稳 | 基本锁定，近期可成交 |
| 可跟 | 有兴趣，需要持续跟进 |
| 观望 | 不排斥但还在犹豫 |
| 冷淡 | 暂时没戏 |

## 5. 手递手式记录体系

### 邀约备注
- 谁写：邀约人（T0/T1）
- 时机：创建客户档案时填一次
- 内容：怎么认识、什么场景、初步印象、关键背景、来源渠道

### 跟进记录（DB 驱动）
- 谁写：邀约人和负责人都可以
- 时机：每次沟通后写一条
- 内容：

| 字段 | 类型 | 说明 |
|---|---|---|
| `customer_id` | BIGINT | 关联客户 |
| `user_id` | BIGINT | 写记录的人 |
| `progress` | VARCHAR(16) | 本次沟通所处的进度 |
| `content` | TEXT | 沟通内容 |
| `feedback` | VARCHAR(500) | 学生反馈 |
| `next_action` | VARCHAR(255) | 下一步动作 |
| `created_at` | DATETIME | 时间戳 |

## 7. 用户体系

| `username` | VARCHAR(64) | 登录账号 |
| `password_hash` | VARCHAR(255) | 密码 |
| `real_name` | VARCHAR(64) | 真实姓名 |
| `level` | VARCHAR(4) | 职级：T0 / T1 / T2 / T3 |
| `manager_user_id` | BIGINT | 直属上级，T3 为空 |
| `status` | VARCHAR(16) | 在职 / 离职 |

### 数据权限规则

```
查询直属上级：SELECT * FROM users WHERE id = 我的 manager_user_id
查询同级同事：SELECT * FROM users WHERE level = 我的level AND manager_user_id = 我的manager_user_id
查询下级：    SELECT * FROM users WHERE manager_user_id = 我的id
查询下下级：  递归往下查 manager_user_id 链
```

## 9. 日报 / 周报 — 方案B 数据联动

### 设计思路

系统自动拉取当天的客户和任务数据，人只需要写三段文字。不增加负担，数据不会造假。

### 9.1 日报表 `daily_reports`

```
daily_reports
├── report_date       DATE          日期
├── user_id           BIGINT        谁写的
│
├── 自动聚合：
│   - new_customer_count       INT   今日新增客户数
│   - follow_up_count          INT   今日跟进沟通次数
│   - progress_advance_count   INT   今日进度推进数（职规→技术栈 等）
│   - task_completed_count     INT   今日完成任务数
│   - progress_details         TEXT  进度推进明细（JSON）
│   - task_completed_details   TEXT  完成任务明细（JSON）
│
├── 人手写：
│   - summary          TEXT          今日工作小结
│   - issues           TEXT          遇到的问题
│   - tomorrow_plan    TEXT          明天计划
│
├── created_at
├── updated_at
│
└── UNIQUE (report_date, user_id)    每人每天一条
```

### 9.2 周报表 `weekly_reports`

```
weekly_reports
├── week_key          VARCHAR(8)    2026-W21
│   week_start        DATE          周起始日
│   week_end          DATE          周结束日
├── user_id           BIGINT
│
├── 自动聚合：
│   - new_customer_count        INT   本周新增客户数
│   - follow_up_count           INT   本周跟进总次数
│   - progress_advance_count    INT   本周进度推进数
│   - converted_count           INT   本周课程达成数（进度到课程+意向很稳）
│   - task_completed_count      INT   本周完成任务数
│   - progress_details          TEXT  进度推进明细
│   - converted_details         TEXT  课程达成明细
│   - task_completed_details    TEXT  完成任务明细
│
├── 人手写：
│   - summary           TEXT          本周总结
│   - issues            TEXT          问题复盘
│   - next_week_plan    TEXT          下周计划
│
├── created_at
├── updated_at
│
└── UNIQUE (week_key, user_id)
```

### 9.3 自动聚合的查询逻辑

| 指标 | SQL 思路 |
|---|---|
| 今日新增客户 | `COUNT(*) FROM customers WHERE DATE(created_at) = CURDATE() AND (inviter_user_id=我 OR owner_user_id=我)` |
| 今日跟进次数 | `COUNT(*) FROM customer_follow_ups WHERE DATE(created_at) = CURDATE() AND user_id = 我` |
| 今日进度推进 | 对比今日写的跟进记录里 progress 和客户当前 progress，发生了变化的算推进 |
| 今日完成任务 | `COUNT(*) FROM task_boards WHERE status='已完成' AND DATE(updated_at) = CURDATE() AND owner_user_id=我` |
| 本周课程达成 | `COUNT(*) FROM customers WHERE progress='课程' AND intent='很稳' AND updated_at BETWEEN 周一 AND 周日` |

### 9.4 前端交互流程

```
打开日报页
    │
    ├── 今天还没写
    │   ├── 系统自动拉聚合数据，填入数字区域（只读）
    │   ├── 显示 "今日新增2个客户，跟进5次，推进1个进度"
    │   ├── 人填写 小结 / 问题 / 明天计划 三个文本框
    │   └── 提交
    │
    ├── 今天已经写了
    │   ├── 展示已提交的完整日报
    │   ├── 自动数据区 + 人手写内容
    │   └── 允许修改（当天内）
    │
    └── 看历史
        ├── 日历选日期
        └── 展示某一天的日报
```

### 9.5 可见范围

```
T3：能看所有人的日报/周报
T2：能看自己 + 直属下级（T1）+ 间接下级（T0）的
T1：能看自己 + 直属下级（T0）的
T0：只能看自己的
```

同一人同一天只能有一条日报，同一个人同一周只能有一条周报。修改只允许当天/当周内。

## 10. 攻略区

替代旧话术库，从"管理员维护模板库"变成"内部协作知识库"。

### 数据结构

```
topics (问题)
├── title              VARCHAR(255)   标题（"如何加到新生？"）
├── description        TEXT           详细描述
├── category           VARCHAR(16)    分类：邀约 / 沟通 / 跟进 / 转化 / 其他
├── author_user_id     BIGINT         提问人
├── solved             TINYINT(1)     是否已解决
├── created_at / updated_at

topic_replies (回答)
├── topic_id           BIGINT         关联问题
├── user_id            BIGINT         回答人
├── content            TEXT           方案内容
├── is_solution        TINYINT(1)     是否被标记为最佳方案
├── created_at
```

### 特点

- 人人可以提问、人人可以出方案
- 提问人可以标记"已解决"和"最佳方案"
- 按分类筛选，支持搜索
- 点赞/收藏/分类订阅 → P2 再做

### 可见范围：全员可见
