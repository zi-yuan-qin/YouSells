## 二、FE-002 登录页联调收口

### 目标
把登录页从"骨架能跑"做到"正式可用"。

### 修改文件（1 个）

#### `src/views/login/LoginView.vue`

**当前问题：**
1. 没有表单校验，空着也能提交
2. 按钮 loading 时没 disabled，可重复点击

**改动点：**

| 项 | 内容 |
|---|---|
| 引入 | 新增 `ref`、`FormInstance`、`FormRules` 从 element-plus |
| 表单 ref | `const formRef = ref<FormInstance>()` |
| 校验规则 | `rules: FormRules = { username: [{ required: true, message: "请输入账号", trigger: "blur" }], password: [{ required: true, message: "请输入密码", trigger: "blur" }] }` |
| 提交前 | `await formRef.value?.validate()`，校验不过直接 return |
| 按钮 | 加 `:disabled="authStore.loading"` |
| el-form | 加 `ref="formRef"` `:model="form"` `:rules="rules"` |
| el-form-item | 加 `prop="username"` 和 `prop="password"` |

### 验收标准
- [ ] 账号或密码为空时点登录，标红提示，不发请求
- [ ] loading 期间按钮 disabled，无法重复点击
- [ ] 登录成功跳转 `/dashboard`
- [ ] 登录失败弹错误提示
- [ ] token 失效自动跳回登录页
- [ ] `npm run build` 不报错

## 三、FE-003 首页看板组件化

### 目标
把 `DashboardView.vue` 内的统计卡片、任务列表、客户列表拆成独立组件，补 loading / empty / error 三态。

### 新建文件（3 个）

#### 1. `src/components/dashboard/DashboardStatCards.vue`

**职责：** 展示四个统计数字卡片

| Props | 类型 | 说明 |
|---|---|---|
| `stats` | `{ label: string; value: number }[]` | 统计项数组 |
| `loading` | `boolean` | 加载中 |

**状态处理：**


#### 2. `src/components/dashboard/DashboardTaskList.vue`

**职责：** 展示今日公共安排列表

| Props | 类型 | 说明 |
|---|---|---|
| `tasks` | `DashboardTaskReminder[]` | 任务数组 |
| `loading` | `boolean` | 加载中 |

**状态处理：**
- 正常：ul > li 列表
- loading：显示 "加载中..."

**模板结构：**
```
.list-card

  v-if loading   → "加载中..."
  v-else-if 空   → "暂无今日安排"
  v-else         → ul > li 循环
```

**单条格式：** `{{ task.taskTitle }}｜{{ task.status }}｜{{ task.ownerDisplayName }}｜截止 {{ task.dueAt }}`

## 四、FE-004 客户总表组件化与筛选区

### 目标
补筛选栏、分页、标签面板，拆分表格，支持行点击跳详情。

### 新建文件（3 个）

#### 1. `src/components/customer-list/CustomerFilterBar.vue`

**职责：** 筛选条件输入 + 搜索/重置

| Props | 类型 | 说明 |
|---|---|---|
| `modelValue` | `CustomerQuery` | 当前查询参数 |
| `loading` | `boolean` | 加载中 |

| Emits | 参数 | 说明 |
|---|---|---|
| `update:modelValue` | `CustomerQuery` | 查询参数变化 |
| `search` | 无 | 点搜索 |
| `reset` | 无 | 点重置 |

**筛选项：**

| 字段 | 组件 | 说明 |
|---|---|---|
| keyword | `el-input` | 搜索客户昵称/编号，支持回车搜索 |
| intentLevel | `el-select` | 全部 / A / B / C / D |
| currentStage | `el-select` | 全部 / 首轮沟通 / 培育中 / 高意向 / 待成交 / 已成交 / 暂停 / 转考 |
| sourcePlatform | `el-input` | 来源平台，支持回车搜索 |
| - | `el-button` | 搜索（primary） |
| - | `el-button` | 重置 |

**逻辑：**
- 搜索时自动 `page = 1`，emit `update:modelValue` + `search`
- 重置时清空所有条件，page 回到 1，emit `update:modelValue` + `reset`

---

#### 2. `src/components/customer-list/CustomerTable.vue`

**职责：** 客户表格 + 分页

| Props | 类型 | 说明 |
|---|---|---|
| `customers` | `CustomerListItem[]` | 客户数据 |
| `loading` | `boolean` | 加载中 |
| `total` | `number` | 总数 |
| `page` | `number` | 当前页 |
| `pageSize` | `number` | 每页条数 |

| Emits | 参数 | 说明 |
|---|---|---|
| `page-change` | `number` | 切换页码 |
| `row-click` | `CustomerListItem` | 点击行 |

**表格列：**

| 列 | prop | 宽度 |
|---|---|---|
| 客户编号 | `customerCode` | min-width 150 |
| 昵称 | `nickname` | min-width 120 |
| 来源平台 | `sourcePlatform` | min-width 110 |
| 意向等级 | `intentLevel` | width 100 |
| 当前阶段 | `currentStage` | min-width 140 |
| 负责人 | `ownerDisplayName` | width 110 |
| 下次跟进 | `nextFollowAt` | min-width 170 |
| 标签 | 自定模板 | min-width 150 |

**标签列：** 用 `el-tag` 渲染，`size="small"` `effect="plain"`，没标签时显示灰色"无"

**分页：** `el-pagination`，`layout="total, prev, pager, next"`，total > 0 才显示

**行点击：** `highlight-current-row`，`@row-click` 触发 emit

---

## 五、接口数据结构（参考）

### `GET /api/dashboard/overview` → `DashboardOverview`

```ts
{
  todayPendingFollowCount: number;
  overdueCustomerCount: number;
  recentNewCustomerCount: number;
  highIntentCustomerCount: number;
  todayTasks: { taskId, taskTitle, status, ownerDisplayName, dueAt }[];
  focusCustomers: { customerId, nickname, intentLevel, currentStage, nextFollowAt }[];
}
```

### `GET /api/customers?page=&pageSize=&keyword=&intentLevel=&currentStage=&sourcePlatform=` → `PageResponse<CustomerListItem>`

```ts
{
  list: { id, customerCode, nickname, customerType, sourcePlatform, intentLevel, currentStage, ownerDisplayName, lastContactAt, nextFollowAt, tags }[];
  page, pageSize, total
}
```

### `GET /api/customers/tags` → `CustomerTag[]`

```ts
{ id, tagName, tagType, tagColor }[]
```
