## 验收过程中定位并修复的问题

### 1. MySQL 8.4 参数兼容问题

现象：

- `yousells-mysql` 容器反复重启

原因：

- `docker-compose.yml` 中的 `--default-authentication-plugin=mysql_native_password` 不再适用于当前 `MySQL 8.4`

处理：

- 删除该启动参数
- 清理旧数据卷后重新初始化容器

结果：

- `MySQL` 容器恢复正常
- 初始化 SQL 成功执行

### 2. dev 环境端口覆盖旧值

现象：

- 后端虽然启动成功，但健康检查报错
- 日志中出现：
  - `Access denied for user 'root'@'localhost'`
  - `Unable to connect to localhost:6379`

原因：

- `application-dev.yml` 仍覆盖使用旧端口：
  - `MySQL: 3306`
  - `Redis: 6379`

处理：

- 将 `application-dev.yml` 更新为：
  - `MySQL: 13306`
  - `Redis: 16379`
- 停掉旧后端实例并重新启动

结果：

- 后端切换到当前容器环境正常运行
