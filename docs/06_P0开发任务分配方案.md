# P0 开发任务分配方案

## 1. P0 目标

`YouSells` 第一阶段先打通这条主流程：

`登录 -> 首页看板 -> 客户总表 -> 客户详情 -> 跟进记录 -> 公共安排 -> 日报/周报 -> 话术库`

第一阶段目标不是做完所有功能，而是把平台的基础能力、业务主流程、测试规范先跑起来。

## 5. 开发顺序建议

推荐顺序：

1. 基础设施与规范
2. 权限与登录基础
3. 客户模块
4. 跟进记录模块
5. 首页看板
6. 公共安排模块
7. 日报 / 周报模块
8. 话术库模块
9. 整体联调与验收

## 6. 当前每个人必须遵守的规则

- 开发前先确认需求和边界
- 开发时同步记录任务进度
- 模块完成后立刻补自动化测试
- 有页面影响的改动必须提醒联调
- 模块交付时不能只交功能代码，必须带测试和验证说明
