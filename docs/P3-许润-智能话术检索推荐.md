## 一、模块目标

在写跟进时，AI 根据当前客户上下文自动推荐最匹配的话术。同时增强话术库搜索能力——输入自然语言问题（如"怎么跟冷淡的客户破冰"），AI 语义匹配相关话术。

1. **跟进时推荐**：在 FollowUpCreateForm 旁新增"AI 话术推荐"侧栏
2. **语义搜索**：话术库搜索框支持自然语言查询
3. **一键引用**：推荐的话术可一键填入跟进内容

## 二、前后端接口契约

### 2.1 POST `/api/scripts/recommend`

**请求体：**
```json
{
  "customerId": 1,
  "context": {
    "progress": "技术栈",
    "intent": "可跟",
    "latestFollowUpContent": "客户表示对Python后端开发感兴趣，但担心学习难度"
  }
}
```

**TypeScript 类型：**
```ts
// src/types/script.ts 中扩展
export interface ScriptRecommendRequest {
  customerId: number;
  context: {
    progress: string;
    intent: string;
    latestFollowUpContent?: string;
  };
}

export interface ScriptRecommendItem {
  script: ScriptItem;
  relevanceScore: number;      // 0-100
  relevanceReason: string;     // "该话术适用于技术栈阶段的客户"
}

// src/api/script.ts 中新增
export async function recommendScripts(data: ScriptRecommendRequest): Promise<ScriptRecommendItem[]> {
  const response = await request.post<ApiResponse<ScriptRecommendItem[]>>("/scripts/recommend", data);
  return response.data.data;
}
```

**响应示例：**

### 2.2 GET `/api/scripts/search`

**查询参数：** `keyword`（自然语言查询字符串）、`page`、`pageSize`

**现有行为：** 基于 SQL LIKE 进行关键词匹配
**P3 增强：** 支持自然语言查询时调 AI 进行语义理解和匹配排序

当 `keyword` 包含自然语言特征（问句、超过一定长度）时，后端将 keyword 和话术库交给 LLM，让 LLM 选出最相关的 N 条。

**响应格式不变**——兼容现有前端分页列表。

### 3.2 话术推荐核心逻辑（Qdrant 向量检索 + LLM 排序）

```
1. 收到请求 { customerId, context }
2. 将客户上下文组装为查询文本：
   query = "客户进度：" + progress + "，意向：" + intent
   if (latestFollowUpContent) query += "，最近跟进：" + latestFollowUpContent
3. 调 EmbeddingService.embed(query) → 查询向量
4. 调 QdrantClient.search("script_vectors", queryVector, limit=10) → 召回 Top10 候选话术
5. 将候选话术 + 客户上下文 → 组装 Prompt → 调 LLM 二次排序
   - LLM 从 10 条候选中选出最匹配的 3-5 条
   - 输出 relevanceScore(0-100) + relevanceReason
6. 用 LLM 返回的 id 查数据库 → 组装 ScriptRecommendItem → 返回
```

**为什么用 Qdrant + LLM 两步：**
- Qdrant 先用语义相似度快速召回候选（毫秒级），避免全量话术传给 LLM
- LLM 再用上下文做精细化排序和理由生成，结果更精准
- 大幅降低 token 消耗（10 条候选 vs 全量 200 条）

### 3.3 语义搜索核心逻辑（Qdrant 增强）

```
1. 收到搜索请求 keyword，判断查询类型：
   a. 短关键词（<=10字符且无问号/疑问词）→ 走原有 SQL LIKE 逻辑（向后兼容）
   b. 自然语言查询（长度>10或包含"怎么"/"如何"/问号）→ 走语义搜索
2. 语义搜索流程：
   a. 调 EmbeddingService.embed(keyword) → 查询向量
   b. 调 QdrantClient.search("script_vectors", queryVector, limit=pageSize, offset)
   c. 用返回的 id 列表 + score 去数据库查完整话术 → 按 score 排序
3. 返回分页结果（格式不变，兼容前端）
```

### 3.4 话术 CRUD 时同步更新 Qdrant

```
1. 话术新建 → 组装文本（title + "\n" + content）→ EmbeddingService.embed() → QdrantClient.upsert()
2. 话术编辑 → QdrantClient.update()（同一向量 id）
3. 话术删除 → QdrantClient.delete()
```

队长 `EmbeddingService` 提供 `syncScriptVector(scriptId, title, content)` 便捷方法，直接调用即可。

### 3.5 Prompt 设计提示（LLM 排序阶段）

System prompt：
> "你是话术库检索专家。根据当前客户的情况和沟通上下文，从话术库中挑选最适合这个场景的话术。评估标准：
> 1. 话术内容与客户当前进度的匹配度
> 2. 话术风格与客户沟通场景的适配性
> 3. 话术是否解决了当前沟通中的具体障碍
> 返回 JSON 数组，每项包含 scriptId、relevanceScore(0-100)、relevanceReason。"

### 3.6 测试场景

| 场景 | 输入 | 期望输出 |
|---|---|---|
| 正常推荐 | customerId=1, 有完整上下文 | Qdrant 召回 + LLM 排序后返回 3-5 条，每项有评分和理由 |
| 话术库为空 | Qdrant collection 为空 | 返回空数组，不调 LLM |
| 客户无跟进记录 | customerId=3, 0条跟进 | 基于进度和意向做 query embedding，正常召回 |
| 语义搜索：自然语言 | keyword="怎么跟冷淡的客户破冰" | Qdrant 向量检索 → 返回语义相关话术 |
| 语义搜索：关键词 | keyword="Python" | 走原 SQL LIKE 逻辑（短关键词不调 Qdrant） |
| 话术 CRUD 同步 | 新建/编辑/删除话术 | Qdrant 向量同步更新 |

**Template 结构（伪代码）：**
```html
<el-drawer v-model="visible" title="AI 话术推荐" direction="rtl" size="400px">
  <!-- 加载中 -->
  <div v-if="loading" class="loading-state">
    <el-icon class="is-loading"><Loading /></el-icon>
    <p>AI 正在分析客户情况，为你推荐最合适的话术...</p>
  </div>

  <!-- 推荐列表 -->
  <div v-else-if="recommendations.length > 0" class="recommend-list">
    <div v-for="item in recommendations" :key="item.script.id" class="recommend-item">
      <div class="item-header">
        <el-tag size="small">{{ item.script.category }}</el-tag>
        <span class="score">匹配度 {{ item.relevanceScore }}%</span>
      </div>
      <h4>{{ item.script.title }}</h4>
      <p class="item-reason">{{ item.relevanceReason }}</p>
      <div class="item-content">{{ item.script.content }}</div>
      <el-button type="primary" size="small" @click="$emit('insert', item.script.content)">
        引用此话术
      </el-button>
    </div>
  </div>

  <!-- 无推荐 -->
  <EmptyState v-else title="暂无推荐" description="话术库中暂未找到匹配当前场景的话术" />

  <!-- 刷新 -->
  <template #footer>
    <el-button :loading="loading" @click="loadRecommendations">换一批</el-button>
  </template>
</el-drawer>
```

**状态覆盖：**

| 状态 | 展示 |
|---|---|
| 加载中 | 骨架屏 + "AI 分析中..." |
| 有推荐 | 推荐列表，每项有评分/理由/内容/引用按钮 |
| 无推荐 | EmptyState |

## 六、遇到问题怎么办

1. **Qdrant 环境**：队长 LEAD-P3-002 已搭好 docker-compose + EmbeddingService，直接 `@Autowired` 注入即可
2. **话术库数据量太小（<20条）**：向量检索效果可能弱，可以先丰富话术库种子数据
3. **推荐不够精准**：问题出在 LLM 排序 Prompt 而非 Qdrant 召回，调整 Prompt 强调匹配标准即可
4. **Qdrant 连接失败**：队长已在 EmbeddingService 内置重试+超时，如果持续不可用会抛异常，前端展示降级提示
5. **前端侧栏太窄显示不完整**：用 el-drawer 可调整宽度，建议 400-480px
6. **卡住超过半天**：群里说明，@梓源
